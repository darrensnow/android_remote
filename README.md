# Android Remote

这是一个基于 Android 的远程协助示例项目，仓库中包含两个独立应用：

- `Application2_service`：服务端，负责屏幕采集、音频采集、局域网发现、无障碍手势执行和保活。
- `Application2_clien`：客户端，负责画面接收、音频播放、远程手势、远程返回和连接管理。

说明：目录名 `Application2_clien` 保留了历史命名，实际它就是客户端工程。

- 服务端增加了音频通道，并优先尝试系统音频捕获，低版本或不支持场景下自动回退到麦克风采集。
- 服务端和客户端增加了局域网自动发现。

- 无障碍未开启时，服务端会在启动前给出提示并跳转到系统设置页。
- 客户端断连时的 UI 线程崩溃已经修复。
- 画面清晰度已较旧版本提升。

- 新增 NAS 可部署 TCP 中继脚本，服务端和客户端都支持公网中继连接。

## 主要功能

- 服务端通过 MediaProjection 采集屏幕并推送给客户端。
- 客户端支持远程滑动、点击、远程返回。
- 客户端支持双指缩放、保持缩放状态和“重置画面”。
- 客户端连接后可在“高清 / 省流”之间动态切换视频清晰度。
- 服务端可通过无障碍服务执行远程手势与全局返回。
- 支持局域网发现，无需手动输入 IP 即可尝试发现服务端。
- 支持 H.264 视频解码显示，客户端直接在 `TextureView` 上渲染。
- 支持音频通道，优先播放服务端系统音频捕获数据，不支持时回退为麦克风采集的 PCM 音频。
- 支持共享口令鉴权。
- 支持自建 NAS 中继模式，双端都可主动连接公网中继，不依赖局域网或额外端口映射。
- 客户端连接成功后顶部控制面板会自动收起，可点击“显示控制”再次展开，避免遮挡远程画面顶部触摸区域。
- 客户端连接成功后顶部和底部控制面板都会自动收起；悬浮的“显示控制”按钮可拖到屏幕任意位置，再用于展开或收起全部控制面板。
- 中文界面，适合直接安装测试。

## 端口说明

- 直连视频与控制端口：`8888`
- 直连音频端口：`8889`
- 局域网发现端口：`8891`（UDP）
- NAS 中继默认端口：`9000`

## 构建方式

两个工程都已经带有自己的 Gradle Wrapper，可以分别构建：

```powershell
cd Application2_service
./gradlew.bat assembleDebug

cd ..\Application2_clien
./gradlew.bat assembleDebug
```

Windows 下也可以直接双击或调用各自目录中的 `gradlew.bat`。

生成后的 APK 路径：

- `Application2_service/app/build/outputs/apk/debug/app-debug.apk`
- `Application2_clien/app/build/outputs/apk/debug/app-debug.apk`

如果需要部署公网中继脚本，可直接运行：

```bash
python3 nas_relay_server.py --host 0.0.0.0 --port 9000
```

脚本位置：`android_remote/nas_relay_server.py`

## 使用步骤

### 1. 安装服务端

将 `Application2_service` 的 APK 安装到被控设备上，首次运行时需要完成：

- 屏幕录制授权
- 无障碍服务开启
- 录音权限授权（系统音频捕获或麦克风回退都需要）

然后根据你的网络方式选择其一：

- 局域网或端口映射直连：保持“NAS 中继模式”关闭，点击“启动服务”。
- 公网 NAS 中继：先在 NAS/公网服务器上启动 `nas_relay_server.py`，然后在服务端勾选“NAS 中继模式”，填写 NAS 地址、端口和房间号，再点击“启动服务”。

界面会显示本机地址、端口信息以及当前的中继配置提示。

### 2. 如需公网，通过 NAS 部署中继

在 NAS、云主机或其他公网 Linux 主机上准备 Python 3，然后运行：

```bash
python3 nas_relay_server.py --host 0.0.0.0 --port 9000
```

如果 NAS 有防火墙，请放通你设置的中继端口，例如 `9000/TCP`。

中继连接规则：

- 服务端和客户端都填写同一个 NAS 地址与端口。
- 服务端和客户端都填写同一个房间号。
- 如果设置了鉴权口令，两端口令也必须一致。

### 3. 安装客户端

将 `Application2_clien` 的 APK 安装到控制端设备上。客户端顶部已拆分为两个独立功能按钮：`局域网连接` 和 `公网连接`。

局域网连接：

- 直接输入服务端地址和端口
- 点击“发现服务端”进行局域网自动发现

公网连接：

- 点击顶部“公网连接”按钮切换模式
- 将地址和端口填写为 NAS 中继地址与端口
- 将房间号填写为与服务端一致的值
- 如果服务端设置了鉴权口令，客户端也要填写相同口令

连接成功后即可观看画面、发送滑动手势、发送远程返回，并可使用“重置画面”恢复默认缩放。

连接后的控制说明：

- 底部 `高清` / `省流` 按钮可动态切换当前视频清晰度。
- 顶部和底部控制面板会在连接成功后自动收起，减少对远程触摸区域的遮挡。
- `显示控制` 按钮可以拖动到屏幕任意位置。
- 如果需要修改地址、模式、房间号、切换清晰度、主动断开或发送返回，可点击 `显示控制` 重新展开全部控制面板。

## 重要前提

- 客户端和服务端处于同一局域网时，局域网发现功能最方便。
- 无障碍服务不是普通危险权限，Android 不支持像麦克风权限那样直接弹框授权，必须由用户在系统设置页手动开启。
- 服务端必须先完成屏幕录制授权，才能真正开始推流。
- 公网连接模式下，双方都必须能主动访问 NAS 中继端口。
- 公网连接模式不使用局域网发现按钮；切回“局域网连接”后才显示发现入口。
- 高清模式更适合局域网下看文字和细节；省流模式更适合公网或带宽波动较大的场景。

## 当前限制

- 当前视频编码已切到 H.264（AVC），尚未升级到 H.265 / HEVC。
- 系统音频捕获依赖 Android 10+ 与目标应用允许被捕获，不满足条件时会自动回退到麦克风输入。
- 当前 NAS 中继是自建的原始 TCP 中继，不是 WebRTC/TURN 方案；优点是实现简单，缺点是没有自适应码率、NAT 探测和更复杂的会话管理。
- 当前公网方案以自建 NAS 中继为主，如果需要更强穿透能力、带宽控制和断线恢复，还需要继续演进信令、重连和监控机制。
- 项目主要用于学习和实验性质的远程协助实现，不建议直接作为生产级远控方案使用。

## 参考说明

如果你准备继续优化，优先级通常是：

1. 将当前 H.264 编码链路继续升级为更稳的动态码率、关键帧恢复和更低延迟参数。
2. 为 NAS 中继补充更完整的会话管理、在线状态、断线重连和审计日志。
3. 补充更完整的公网鉴权、错误提示和后台保活策略。
